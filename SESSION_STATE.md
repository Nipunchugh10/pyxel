## Key Technical Notes

1. **Python environment:** MSYS2 UCRT64 Python 3.12.10 at `C:\msys64\ucrt64\bin\python.exe`
2. **Package manager:** Use `C:\msys64\usr\bin\pacman.exe` for installing Python packages (not pip directly — externally managed environment)
3. **Matplotlib backend:** `renderer.py` auto-detects Jupyter vs CLI and sets `inline` or `agg` accordingly
4. **Stub pattern rendering:** Remaining stubs use `_StubMixin` that renders a dark card with pattern name + "⏳ Coming Soon"
5. **Git remote:** `origin` → `https://github.com/Nipunchugh10/pyxel.git` (configured and pushing to `main`)

### UI Architecture — Critical Notes (after bug-fix session 2026-05-13)

6. **Export capture:** `renderer._fig` is set by `BasePattern._create_figure()` and remains a valid Python object after `plt.close(fig)` removes it from pyplot's manager. `export_png()` calls `figure.savefig()` on this object directly — no need to re-open the figure.

7. **Single image per render:** Every `render()` method calls `plt.show()` then `plt.close(fig)`. The `plt.show()` call — captured inside `with render_output:` — is the **one and only** display event. Do NOT add any `display(fig)` or `display(IPImage(...))` call after `renderer.render()` returns; that would create a second image.

8. **Controls are cached — never recreate them:** `_get_controls(name)` in the notebook returns the same widget instances every time (stored in `_controls_cache`). Both `_on_pattern_change` (display) and `_on_render` (value read) use this cache. If you call `renderer.get_controls()` directly anywhere, you get fresh widgets with default values and lose the user's slider adjustments.

9. **Controls panel uses VBox, not Output:** `pattern_controls_area` is a `widgets.VBox`. Swapping controls is done by setting `.children` atomically — no `clear_output()`/`display()` calls needed. This avoids VS Code Jupyter's `clear_output(wait=True)` race condition that caused doubled controls.
